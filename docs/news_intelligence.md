# CIVID — News Intelligence Layer

The news intelligence table (`data/phase<N>_*/news_intelligence.csv`, schema
`schema/news_intelligence_schema.json`) is an **authentic, source-linked** summary layer
built from the dataset — not free-text news scraping.

## What it contains

- **Computed aggregate metrics** (generated by `scripts/generate_news_intelligence.py`):
  `total_killed`, `children_killed`, `women_killed`, `doctors_killed`,
  `journalists_killed`, `commanders_killed`, `arrests`, `detentions`.
  Each row records the `metric`, `value`, `news_category`, a `citation_text` describing how
  it was derived, the supporting `source_id`(s) in `verified_by`, and `verification_status`.
- **Curated news stories** (optional, human-added): rows with `news_headline`,
  `news_summary`, `news_source_url`, `news_date`, `news_category`. These must cite a real
  source — never invented.

## Categories (controlled)

`casualties`, `child impact`, `medical workers`, `leaders`, `arrests`, `humanitarian update`.

## Rules

1. Every row has `citation_text` **and** `verified_by` (≥1 source id). The validator rejects
   rows missing either.
2. Numbers are derived from verified rows only; `unverified` rows are excluded from aggregates.
3. `women_killed` is intentionally `0` until a per-victim fatality link is modeled — we do not
   guess it.
4. The HTML dashboard renders these as the "aggregate intelligence" panel + news section, each
   number traceable back to `sources.csv`.
5. Regenerate after any data change: `python scripts/generate_news_intelligence.py`.
